# Contributing to Kaydelish.HTML

Thanks for wanting to contribute! We welcome bug reports, feature requests, documentation fixes, and code contributions.

How to contribute
- Report bugs: open an issue using the Bug Report template.
- Request features: open an issue using the Feature Request template.
- Submit code: fork the repo, create a branch, and open a pull request.

Branching and pull requests
- Use a short, descriptive branch name: `fix/issue-123-short-desc` or `feat/add-order-page`.
- Rebase or merge the latest `main` before opening a PR.
- Include a short description and link any related issues in the PR body.

Commit messages
- Keep messages clear and imperative: `Add X`, `Fix Y`.
- Reference issue numbers where applicable: `Fixes #12`.

Code style
- Follow existing HTML, CSS and JavaScript style in the repository.
- Run any linters or formatters (if present) before submitting the PR.

Testing
- Describe manual testing steps in the PR if no automated tests exist.

Review process
- PRs will be reviewed by maintainers; be responsive to review feedback.

Local development
- There is no enforced build system; open the pages in a browser or use a simple static server:

```bash
python3 -m http.server 8000
# or
npx http-server
```
